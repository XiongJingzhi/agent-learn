## 📚 核心概念

### 概念 1：为什么需要环境检查？

> **类比**：就像一个人**去医院体检**一样，环境检查可以提前发现问题，避免后续出现更严重的问题。

**环境检查的重要性**：

1. ✅ **提前发现问题**：提前发现依赖缺失、版本不匹配等问题
2. ✅ **避免运行时错误**：避免在运行时才发现依赖缺失等问题
3. ✅ **确保兼容性**：确保所有依赖的版本都兼容
4. ✅ **提高开发效率**：避免因为环境问题浪费开发时间

---

### 概念 2：环境检查的步骤

#### 步骤 1：检查 Python 版本

> **类比**：就像检查**手机系统版本**一样，检查 Python 版本确保兼容性。

**检查命令**：
```bash
python --version
# 或
python3 --version
```

**预期版本**：Python 3.8+

---

#### 步骤 2：检查 pip 版本

> **类比**：就像检查**应用商店版本**一样，检查 pip 版本确保兼容性。

**检查命令**：
```bash
pip --version
```

**预期版本**：pip 21.0+

---

#### 步骤 3：检查依赖安装

> **类比**：就像检查**应用是否已安装**一样，检查依赖是否已安装。

**检查命令**：
```bash
pip list
```

**预期输出**：列出所有已安装的依赖包和版本

---

#### 步骤 4：检查依赖版本

> **类比**：就像检查**应用版本是否符合要求**一样，检查依赖版本是否符合要求。

**检查命令**：
```bash
pip show langchain
pip show langgraph
```

**预期版本**：
- langchain >= 0.1.0
- langgraph >= 0.1.0

---

#### 步骤 5：安装依赖

> **类比**：就像**安装应用**一样，安装缺失的依赖。

**安装命令**：
```bash
pip install -r requirements.txt
```

**预期输出**：成功安装所有依赖

---

### 概念 3：环境配置

#### 1. 配置虚拟环境

> **类比**：就像创建一个**隔离的工作空间**一样，虚拟环境可以隔离不同项目的依赖。

**创建虚拟环境**：
```bash
python -m venv venv

# 激活虚拟环境
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate  # Windows
```

---

#### 2. 配置环境变量

> **类比**：就像**设置工作空间的环境变量**一样，配置环境变量可以让工具找到正确的路径。

**配置环境变量**：
```bash
# Linux/Mac
export OPENAI_API_KEY="your-api-key"

# Windows
set OPENAI_API_KEY="your-api-key"
```

---

#### 3. 配置日志级别

> **类比**：就像**设置手机的日志级别**一样，配置日志级别可以控制日志的详细程度。

**配置日志级别**：
```python
import logging

# 配置日志级别
logging.basicConfig(level=logging.DEBUG)
```
