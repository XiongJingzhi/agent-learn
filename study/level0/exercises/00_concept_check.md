## 🎯 简答题（11-20）

### 第 1 部分：概念解释（11-15）

**Q11. 简单解释 Agent 是什么？**

**答案**: Agent 就像一个聪明的秘书，他不仅能按照指令工作，还能根据情况自主决策如何完成任务。Agent 有四个核心组件：感知（接收信息）、推理（分析情况）、执行（执行决策）、记忆（存储历史）。

**Q12. 简单解释 ReAct 循环的四个阶段。**

**答案**: ReAct 循环的四个阶段是：思考（分析当前情况，制定下一步计划）、行动（执行计划，调用工具或生成回复）、观察（观察行动的结果，更新状态）、循环（根据观察结果决定是否继续下一轮循环）。

**Q13. 简单解释 Agent 和 Chatbot 的区别。**

**答案**: Agent 和 Chatbot 的主要区别是：Agent 能自主决策和适应环境变化，而 Chatbot 只能按照固定的规则或流程执行。Agent 有推理和学习能力，而 Chatbot 没有。

**Q14. 简单解释 Agent 状态的三种类型。**

**答案**: Agent 状态的三种类型是：短期状态（像人的短期记忆，只能记住最近发生的几件事）、中期状态（像人的中期记忆，可以记住一段时间内的事情）、长期状态（像人的长期记忆，可以记住很久以前发生的事情）。

**Q15. 简单解释 Agent 状态更新的三种策略。**

**答案**: Agent 状态更新的三种策略是：直接更新（像直接修改记忆一样，新的信息会覆盖旧的信息）、追加更新（像追加新信息一样，新的信息会追加到旧的后面）、分层更新（像分层次管理记忆一样，短期、中期、长期分开管理）。

### 第 2 部分：应用场景（16-20）

**Q16. 举一个适合使用 Agent 的场景。**

**答案**: 智能客服系统。用户的问题复杂多样，需要 Agent 根据问题自主决策是调用搜索工具、计算器工具，还是直接生成回复。

**Q17. 举一个适合使用 Chatbot 的场景。**

**答案**: 客户服务常见问题解答（FAQ）。用户的问题都是预设的，回复也是预设的，不需要推理和决策。

**Q18. 举一个需要使用记忆的 Agent 场景。**

**答案**: 多轮对话的智能助手。Agent 需要记住用户的姓名、偏好设置、对话历史等信息，才能提供个性化的服务。

**Q19. 举一个需要使用工具的 Agent 场景。**

**答案**: 自动化办公助手。Agent 需要调用各种工具来完成不同的任务，如：发送邮件、创建日历、查询数据库等。

**Q20. 举一个可能失败并需要错误处理的 Agent 场景。**

**答案**: 搜索信息的 Agent。用户查询的信息可能不存在，或者搜索工具可能调用失败，Agent 需要优雅地处理这些错误情况，并给出友好的反馈。
