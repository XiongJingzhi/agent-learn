# Level 4 进阶练习与挑战项目

## 02_intermediate_exercises.md - 进阶练习（30 题）

### 第一部分：实现题（15 题）

1. **实现一个简单的任务生成函数**
   - 输入：目标描述
   - 输出：3-5 个子任务

2. **实现任务优先级排序**
   - 基于重要性和紧迫性
   - 返回排序后的任务列表

3. **实现一个反思函数**
   - 评估执行结果（0-1 分）
   - 生成改进建议

4. **实现混合检索**
   - 结合稠密和稀疏检索
   - alpha=0.6

5. **实现 RRF 融合**
   - 基于排名融合
   - k=60

6. **实现简单的缓存**
   - 精确匹配缓存
   - LRU 淘汰策略

7. **实现成本监控**
   - 跟踪 Token 使用
   - 预算警告

8. **实现异步任务执行**
   - 使用 asyncio
   - 并发数限制

9. **实现两阶段检索**
   - 初始检索 Top-50
   - 重排序返回 Top-10

10. **实现查询重写**
    - 扩展同义词
    - 保持原意

### 第二部分：设计题（10 题）

11. **设计一个自主 Agent 的架构**
    - 包含哪些组件？
    - 组件间如何交互？

12. **设计一个多层缓存系统**
    - L1：精确匹配
    - L2：语义相似
    - L3：LLM 调用

13. **设计一个目标分解策略**
    - 如何确定分解深度？
    - 如何处理依赖关系？

14. **设计一个反思循环**
    - 何时触发反思？
    - 如何验证反思效果？

15. **设计一个混合 RAG 系统**
    - 何时使用向量检索？
    - 何时使用图谱检索？
    - 如何融合结果？

16. **设计一个性能监控系统**
    - 需要监控哪些指标？
    - 如何设置告警？

17. **设计一个 A/B 测试方案**
    - 如何分组？
    - 如何评估结果？

18. **设计一个知识图谱更新策略**
    - 如何处理时效性？
    - 如何验证准确性？

19. **设计一个成本优化方案**
    - 在哪些环节优化？
    - 如何平衡质量和成本？

20. **设计一个自主研究助手**
    - 核心功能是什么？
    - 如何保证质量？

### 第三部分：分析题（5 题）

21. **分析 BabyAGI 和 AutoGPT 的适用场景**
    - BabyAGI 适合什么任务？
    - AutoGPT 适合什么任务？

22. **分析混合检索的权衡**
    - alpha 的选择对结果的影响
    - 何时偏向稠密或稀疏？

23. **分析重排序的成本效益**
    - 什么情况下值得重排序？
    - 如何平衡准确率和成本？

24. **分析缓存的利弊**
    - 缓存能带来多少收益？
    - 有什么风险？

25. **分析自主 Agent 的局限性**
    - 哪些场景不适合自主 Agent？
    - 如何改进？

## 03_challenge_projects.md - 挑战项目

### 项目 1：构建一个 BabyAGI Agent

**目标**：实现一个完整的 BabyAGI 系统

**要求**：
1. 实现任务生成、排序、执行循环
2. 支持自定义工具
3. 记录执行历史
4. 可视化执行过程

**验收标准**：
- [ ] 能够自主完成任务
- [ ] 至少支持 3 种工具
- [ ] 有清晰的日志输出
- [ ] 代码结构清晰

**提示**：
- 从简单的循环开始
- 逐步添加功能
- 测试每个组件

---

### 项目 2：构建混合 RAG 系统

**目标**：实现一个高效的混合检索系统

**要求**：
1. 结合稠密和稀疏检索
2. 实现重排序
3. 支持查询优化
4. 性能监控

**验收标准**：
- [ ] 检索准确率 > 基线 15%
- [ ] 响应时间 < 500ms
- [ ] 支持至少 1000 文档
- [ ] 有完整的测试

**提示**：
- 先实现单一检索
- 再实现混合
- 最后添加重排序

---

### 项目 3：构建自主研究助手

**目标**：集成所学技术，构建完整应用

**要求**：

2. 混合检索相关资料

**验收标准**：
- [ ] 能够完成复杂研究任务
- [ ] 生成高质量报告
- [ ] 成本可控
- [ ] 用户体验良好

**提示**：
- 参考 AutoGPT 架构
- 集成高级 RAG
- 添加性能优化
