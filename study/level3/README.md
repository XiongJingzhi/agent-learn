# Level 3：框架对比与选择

## 学习目标
通过对比学习形成框架迁移能力，能基于场景做框架选型并说明权衡。

## 前置条件
- 完成 Level 2
- 熟悉至少一个 LangGraph 实战案例

## 核心主题
- 架构：框架设计理念与约束对比
- 实现：同题多框架实现与代码对照
- 学习方法：概念映射表、费曼解释、类比复盘
- 测试：功能等价性、性能与开发效率对比

## OpenClaw / OpenCode 信息分区
### 已确认
- OpenClaw：多渠道接入、Gateway、语音/可视化能力导向。
- OpenCode：开源编程助手定位、TUI/LSP/MCP 集成导向。

### 待确认
- OpenClaw 与 OpenCode 的最新稳定接口细节。
- 两者在本仓库学习体系中的固定示例实现边界。

## 实践任务
1. 完成一个“同题在不同框架实现”的对比报告。
2. 建立概念映射：状态、工具、流程、记忆。
3. 给出三个业务场景的框架选型与理由。
4. 编写功能等价性与性能基准测试。

## 完成标准
- 能清晰说明至少三种框架的核心差异。
- 能针对业务约束给出可辩护的选型决策。
- 输出一份可复用的迁移清单。

## 常见误区
- 只比较语法，不比较设计目标和系统边界。
- 未固定评估指标，导致结论不可复现。
- 把“熟悉度”误当“最优方案”。
