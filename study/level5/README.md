# Level 5：多 Agent 协作系统

## 学习目标
掌握多 Agent 协作模式与通信机制，能设计稳定的协作流程与冲突处理策略。

## 前置条件
- 完成 Level 4
- 理解单 Agent 的状态与工具边界

## 核心主题
- 架构：层级式、协作式、竞争式、流水线式
- 实现：角色分工、任务移交、共享上下文
- 学习方法：先定义协作协议，再实现流程
- 测试：协作一致性、消息时序、冲突恢复

## 实践任务
1. 设计并实现一个 3 角色协作系统。
2. 定义角色职责、输入输出协议与失败回退规则。
3. 构建一个端到端协作任务并记录瓶颈。
4. 编写通信与冲突场景测试。

## 完成标准
- 能给出可解释的角色边界与协作协议。
- 多 Agent 在失败与重试场景下可控。
- 具备覆盖关键协作路径的测试集。

## 常见误区
- 角色职责重叠，导致重复推理或冲突。
- 无状态同步协议，难以追踪协作过程。
- 忽视通信成本与上下文膨胀。
