# 挑战项目

> **目标**: 在真实场景中应用 Level 2 知识
> **预计时间**: 3 小时
> **难度**: ⭐⭐⭐⭐

---

## 挑战 1：Research Assistant Agent

### 目标
构建一个能够自主进行研究和总结的 Agent。

### 功能要求

1. **任务规划**：
   - 将研究主题分解为子任务
   - 确定搜索关键词
   - 规划总结结构

2. **执行能力**：
   - 搜索多个信息源
   - 提取关键信息
   - 验证信息准确性

3. **反思优化**：
   - 评估信息的完整性
   - 识别矛盾的信息
   - 补充缺失的内容

### 验收标准

- [ ] 能够处理复杂的研究主题（如"人工智能的发展历史"）
- [ ] 生成结构化的研究报告
- [ ] 引用信息来源
- [ ] 有完整的错误处理

### 示例输入

```
研究主题: "量子计算的基本原理和应用"

期望输出:
1. 研究计划（3-5 个子任务）
2. 每个子任务的执行结果
3. 最终的研究报告（包含引文）
```

### 提示

- 使用 PER 架构
- 实现任务优先级（核心概念优先）
- 使用记忆系统存储已收集的信息
- 实现至少一轮反思优化

## 挑战 2：Multi-Round Conversation Agent

### 目标
构建一个能够处理多轮对话并保持上下文的 Agent。

### 功能要求

1. **记忆管理**：
   - 短期记忆：当前对话
   - 中期记忆：对话摘要
   - 长期记忆：用户偏好

2. **上下文追踪**：
   - 记录对话主题
   - 追踪未解决的问题
   - 识别用户意图变化

3. **智能检索**：
   - 根据当前问题检索相关历史
   - 优先考虑最近的相关对话
   - 跨会话检索用户偏好

### 验收标准

- [ ] 能够处理 10+ 轮对话而不丢失上下文
- [ ] 能够记住用户偏好（至少 3 种）
- [ ] 能够生成准确的对话摘要
- [ ] 检索响应时间 < 100ms

### 示例对话

```
用户: 我喜欢 Python
Agent: 好的，我记住了你喜欢 Python

用户: 推荐一些学习资源
Agent: 既然你喜欢 Python，我推荐...
（根据用户偏好和对话历史给出回答）
```

### 提示

- 实现三层记忆系统
- 使用向量相似度进行检索
- 定期生成对话摘要
- 实现遗忘机制（不重要的信息逐渐淡化）

## 挑战 3：Code Review Agent

### 目标
构建一个能够审查代码并提供改进建议的 Agent。

### 功能要求

1. **代码分析**：
   - 识别潜在 bug
   - 检查代码风格
   - 评估代码复杂度

2. **建议生成**：
   - 提供具体的改进建议
   - 解释问题的原因
   - 给出改进示例

3. **迭代改进**：
   - 根据反馈调整建议
   - 优先处理严重问题
   - 跟踪改进历史

### 验收标准

- [ ] 能够识别至少 5 种常见问题
- [ ] 建议的具体性（不是泛泛而谈）
- [ ] 支持多轮反馈和改进
- [ ] 有清晰的优先级排序

### 示例输入

```python
# 待审查代码
def calculate(a, b, c):
    x = a + b
    y = x * c
    z = y / x
    return z
```

### 期望输出

```
问题识别:
1. 第 4 行: 除以零风险 (严重)
   - 当 x=0 时会导致错误
   - 建议: 添加零值检查

改进建议:
def calculate(a, b, c):
    x = a + b
    y = x * c
    if x != 0:  # 添加检查
        z = y / x
        return z
    else:
        return None  # 或抛出异常
```

### 提示

- 使用 Reflection 推理进行多轮分析
- 实现问题严重性评分
- 使用思维链解释推理过程
- 记录审查历史以便学习

## 挑战 4：故障排查 Agent

### 目标
构建一个能够诊断和解决系统故障的 Agent。

### 功能要求

1. **故障诊断**：
   - 收集系统信息
   - 分析日志文件
   - 识别错误模式

2. **假设验证**：
   - 生成多个可能原因
   - 设计验证步骤
   - 排除不可能原因

3. **解决方案**：
   - 提供解决步骤
   - 评估风险
   - 提供回滚方案

### 验收标准

- [ ] 能够诊断至少 3 种常见故障
- [ ] 使用 ToT 探索多种可能原因
- [ ] 提供可执行的解决步骤
- [ ] 有风险评估和回滚方案

### 示例场景

```
故障现象: "网站响应很慢"

诊断过程:
1. 收集信息（CPU、内存、网络）
2. 生成假设（数据库慢？网络延迟？代码问题？）
3. 验证假设（查询日志、运行测试）
4. 给出解决方案（优化查询、增加缓存等）
```

### 提示

- 使用 ToT 探索多种可能性
- 实现系统信息收集工具
- 使用检查点记录诊断过程
- 支持回滚到之前状态

## 挑战 5：学习规划 Agent

### 目标
构建一个能够为用户制定个性化学习计划的 Agent。

### 功能要求

1. **能力评估**：
   - 评估用户当前水平
   - 识别知识差距
   - 确定学习目标

2. **计划制定**：
   - 分解学习目标
   - 排序学习内容（依赖关系）
   - 估计学习时间

3. **进度跟踪**：
   - 记录学习进度
   - 调整学习计划
   - 提供反馈和建议

### 验收标准

- [ ] 能够评估用户在至少 2 个领域的水平
- [ ] 生成符合逻辑的学习顺序（考虑先修课程）
- [ ] 跟踪学习进度并动态调整
- [ ] 提供个性化的学习建议

### 示例对话

```
用户: 我想学习机器学习

Agent: 让我先了解一下你的背景...

[评估过程]
- 数学基础: 良好
- 编程经验: 中等
- ML 知识: 初学者

[学习计划]
阶段 1 (2 周): 数学基础复习
  - 线性代数
  - 概率统计
  - 微积分基础

阶段 2 (3 周): 编程基础
  - Python 入门
  - NumPy/Pandas
  - 数据可视化

阶段 3 (4 周): ML 核心概念
  ...
```

### 提示

- 使用 DAG 表示学习内容依赖
- 实现动态优先级（根据用户进度调整）
- 使用记忆系统跟踪学习历史
- 实现反馈循环（根据学习效果调整）

## 评分标准

每个挑战项目将从以下维度评分：

### 完成度 (30%)
- 是否实现所有核心功能
- 是否满足验收标准

### 代码质量 (25%)
- 代码结构和可读性
- 错误处理
- 文档注释

### 创新性 (20%)
- 是否有独特的解决方案
- 是否超越基本要求

### 测试覆盖 (15%)
- 是否有充分的测试
- 是否考虑边界情况

### 可扩展性 (10%)
- 代码是否易于扩展
- 是否考虑未来需求

## 完成建议

1. **选择一个挑战**：不要试图完成所有挑战
2. **迭代开发**：先实现基本功能，再逐步完善
3. **记录过程**：记录设计决策和遇到的问题
4. **编写测试**：确保代码质量
5. **寻求反馈**：与他人讨论你的方案
