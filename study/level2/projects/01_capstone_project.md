# Capstone Project: 智能 Planning Agent

> **项目目标**: 构建一个完整的任务规划型 Agent
> **预计时间**: 10-12 小时
> **难度**: ⭐⭐⭐⭐⭐

---

## 项目概述

构建一个能够理解用户复杂需求、制定执行计划、动态调整策略的智能 Planning Agent。

**核心能力**：
1. 理解自然语言任务描述
2. 自动分解任务为 DAG
3. 执行任务并处理失败
4. 反思结果并优化
5. 管理多层次记忆

---

## 项目要求

### 功能要求 (MVP)

#### 1. Planner 组件

**基础功能**：
- [ ] 接受自然语言任务描述
- [ ] 分析任务类型（搜索/计算/分析等）
- [ ] 生成任务 DAG（至少 3 种任务类型）
- [ ] 设置任务优先级

**高级功能**（可选）：
- [ ] 支持嵌套任务分解
- [ ] 动态调整任务顺序
- [ ] 预估任务执行时间

#### 2. Executor 组件

**基础功能**：
- [ ] 按拓扑顺序执行任务
- [ ] 处理任务依赖
- [ ] 记录执行状态
- [ ] 实现基础错误处理

**高级功能**（可选）：
- [ ] 支持并行执行无依赖任务
- [ ] 实现重试机制（指数退避）
- [ ] 支持任务超时
- [ ] 实现检查点和恢复

#### 3. Reflector 组件

**基础功能**：
- [ ] 计算任务完成度
- [ ] 识别失败任务
- [ ] 判断是否需要重规划
- [ ] 生成改进建议

**高级功能**（可选）：
- [ ] 多轮反思优化
- [ ] 学习历史失败模式
- [ ] 自适应调整阈值

#### 4. Memory System

**基础功能**：
- [ ] 短期记忆（当前对话）
- [ ] 中期记忆（对话摘要）
- [ ] 长期记忆（用户偏好）

**高级功能**（可选）：
- [ ] 向量检索
- [ ] 重要性评分
- [ ] 自动记忆合并

## 常见问题

### Q1: 是否必须使用真实的 LLM？

**答**：不是必须的。可以使用 Mock LLM 进行开发和测试。在最终演示时，如果有条件可以接入真实 LLM。

### Q2: 如何处理任务分解的准确性？

**答**：
1. 从简单明确的任务类型开始
2. 使用规则和模板进行分解
3. 逐步引入 LLM 辅助分解
4. 通过反思循环改进分解质量

### Q3: 记忆系统是否必须使用向量数据库？

**答**：不是必须的。MVP 可以使用简单的关键词匹配。向量检索可以作为高级功能实现。

### Q4: 如何测试 Agent 的行为？

**答**：
1. 单元测试：测试各个组件
2. 集成测试：测试组件交互
3. 使用 Mock：隔离外部依赖
4. 场景测试：测试端到端行为
