## 🚀 核心亮点

### 1. Tape-First 架构

- 📝 **完整历史**: 保留完整的会话历史
- 🔍 **易于调试**: 可以追溯所有操作
- 🔄 **可恢复**: 支持从任何点恢复
- 📊 **可分析**: 可以分析用户行为

### 2. 严格的命令边界

- 🎯 **明确性**: 用户清楚知道何时发送命令
- 🛡️ **安全性**: 避免意外的命令执行
- 🤖 **灵活性**: 允许模型生成命令

### 3. 双路由模型

- 🔄 **一致性**: 路由逻辑一致
- 🧠 **智能回退**: 失败命令自动回退
- 📝 **上下文保留**: 保留错误上下文

### 4. 异步优先

- 🚀 **高性能**: 可以并发执行多个操作
- 🔄 **非阻塞**: I/O 操作不会阻塞主循环
- 📊 **可扩展**: 容易扩展到高并发场景

## 🎯 核心概念速查

### Tape（磁带）

**定义**: 仅追加的 JSONL 日志系统，记录会话中的所有事件。

**关键特性**:
- ✅ 仅追加
- ✅ 持久化
- ✅ 可查询
- ✅ 可分支
- ✅ 可归档

### Command（命令）

**定义**: Bub 的基本操作单元，分为内部命令和 Shell 命令。

**命令类型**:
- 内部命令: `,help`, `,tools`, `,tape.info`
- Shell 命令: `,git status`, `,ls -la`
- 自然语言: `hello`, `如何使用 bub`

### Router（路由器）

**定义**: 命令路由和执行的核心组件，负责判断输入是命令还是自然语言。

**路由流程**:
- 输入 → 解析 → 判断 → 执行

### Tool（工具）

**定义**: 可扩展的功能单元，可以是内置工具、内部命令或技能支持。

**工具分类**:
- 内置工具: `fs.read`, `fs.write`
- 内部命令: `help`, `tools`, `tape.info`
- 技能工具: 从 `skills/` 目录加载

### Anchor / Handoff（锚定/移交）

**定义**: 上下文转换机制，用于在会话中创建明确的阶段划分。

**用途**:
- 阶段划分
- 锚点标记
- 上下文切换
- 摘要记录

## 🎉 总结

Bub 是一个设计精良的编码 Agent CLI，具有以下特点：

1. **Tape-First 架构**: 所有操作都记录到磁带上，确保完整的历史记录和可恢复性
2. **严格的命令边界**: 只有以 `,` 开头的行才被视为命令，确保明确性和安全性
3. **双路由模型**: 同样的路由逻辑应用于用户输入和助手输出，确保一致性
4. **异步优先**: 完全异步的执行模型，支持高性能和可扩展性
5. **多通道支持**: 支持 CLI、Telegram、Discord，满足不同的使用场景

这些设计使得 Bub 成为一个**可预测、可检查、可恢复**的编码 Agent，非常适合真实的工程工作流。
